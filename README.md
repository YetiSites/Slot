# Web Slot Machine - 技術仕様・設計詳細ドキュメント

本プロジェクトは、実機のパチスロ（特にジャグラーシリーズ）の挙動をWeb上で詳細に再現するための技術デモです。

## 1. ソフトウェアアーキテクチャ

ES6クラスを用いた役割分担により、高い保守性と拡張性を実現しています。

| クラス名 | 役割 | 主な責務 |
| :--- | :--- | :--- |
| **`SlotGame`** | ゲームエンジン | 状態管理、抽選実行、入賞判定の集約、LocalStorage永続化 |
| **`ReelController`** | リール司令塔 | 3リールの同期制御、スベリコマ数の計算（テーブル参照） |
| **`Reel`** | 個別リール | 回転アニメーション、シンボルの描画とループ処理 |
| **`SoundFX`** | サウンド | Web Audio APIによる効果音・BGMの合成と再生 |

---

## 2. 内部抽選ロジック

「レバーON」の瞬間に、JavaScriptの乱数を用いて「そのプレイの運命（フラグ）」を決定します。

### 抽選アルゴリズム
1. **分母の設定**: 全体を `65,536` (16bit) の範囲として扱います。
2. **テーブル選択**:
   - `NORMAL`: 通常時
   - `BONUS_ACTIVE`: ボーナス内部中（ハズレ、リプレイ、ブドウ、ベルのみ。チェリー等は出現しない）
   - `BONUS_GAME`: ボーナス中（ブドウを超高確率で抽選）
3. **累積重み判定**: 乱数値が各役の `weight` の累積範囲に収まるか順にチェックします。
   - 例: `FREEZE` (64) -> `REPLAY` (9298) -> ... -> `BLANK` (ハズレ)

---

## 3. 出目制御（スベリ制御）の深層

本システムの最も重要なロジックです。停止ボタンが押された瞬間のリール位置と、成立フラグに基づいて停止位置を決定します。

### スベリ計算のステップ
1. **停止ボタン押下**: 押下された「コマ（フレーム）」の位置をキャプチャ。
2. **引き込み（Pull-in）**:
   - `STOP_CONTROL_TABLE` を参照し、成立フラグに対する「ターゲット絵柄」を特定。
   - 最大4コマ（規定のスベリ範囲）の間にターゲット絵柄がある場合、その絵柄が中段（または規定ライン）に止まるようスベリコマ数を算出。
3. **蹴り（Kicking / Safe Slip）**:
   - フラグがハズレの場合、あるいは成立フラグ以外の役が揃ってしまう位置にある場合、それらを「避ける」スベリを行います。
4. **内部ボーナス維持**:
   - 内部でボーナスが成立している場合、目押しミス等で揃えられなくても、内部フラグ（`internalBonus`）は消失せず、次ゲーム以降も維持されます。

## 4. ボーナス・システム詳細

実機の挙動に合わせた特殊な状態遷移を実装しています。

### ボーナス入賞（777 / 77BAR）
- 入賞時は **「リプレイ扱い」** となります（追加のメダル消費なしで次プレイへ）。
- 入賞そのものによる払い出しはありません（古い配当設定 312枚等は廃止）。

### ボーナスゲーム（小役ゲーム）
- **1枚掛け固定**: ボーナス中はメダル1枚でプレイ可能です。
- **ブドウ払い出し**: ボーナス中のブドウ入賞時にメダルが払い出されます。
  - **通常ボーナス**: 24枚払い出し。
  - **プレミアムボーナス (フリーズ経由)**: **72枚払い出し**。
- **規定入賞回数**: 
  - `BIG`: 12回のブドウ入賞で終了。
  - `REG`: 6回のブドウ入賞で終了。

## 5. 演出・サウンド仕様

### GOGO告知
- **先告知**: レバーON時に、内部ボーナス成立時の25%の確率で点灯。
- **後告知**: 全リール停止時に点灯。
- **BLINK状態**: プレミアムフリーズ時は、ランプが高速に点滅し、**ボーナス終了まで点滅が持続** します。

### プレミアムフリーズ
- 内部的に `FREEZE` 役が成立（または開発者モードで指定）した際、レバーONで4秒間のフリーズ（画面暗転・専用SE）が発生します。
- フリーズ後は「プレミアムボーナス」が確定します。

---

## 6. 開発者モード (Developer Mode)

ブラウザ上でフラグを自由に操作し、特定の挙動を検証することが可能です。

- **小役フラグ強制**: 次回のレバーONで成立させるフラグを「BIG」「REG」「BAR揃い」「フリーズ」「チェリー」等から選択できます。
- **内部状態表示**: 現在の成立フラグや、内部ボーナスの有無（内部中フラグ）をリアルタイムで確認できます。

---

## 7. 保存と復元 (LocalStorage)

ブラウザの `localStorage` キー `slot_game_state` に、経済状態やボーナス進捗（`isPremiumBonus` 等を含む）を保存します。これにより、プレイを中断しても次回起動時に全く同じ状態から復元されます。
