# Web Slot Machine - 技術仕様・設計詳細ドキュメント

本プロジェクトは、実機のパチスロ（特にジャグラーシリーズ）の挙動をWeb上で詳細に再現するための技術デモです。

## 1. ソフトウェアアーキテクチャ

ES6クラスを用いた役割分担により、高い保守性と拡張性を実現しています。

| クラス名 | 役割 | 主な責務 |
| :--- | :--- | :--- |
| **`SlotGame`** | ゲームエンジン | 状態管理、抽選実行、入賞判定の集約、LocalStorage永続化 |
| **`ReelController`** | リール司令塔 | 3リールの同期制御、スベリコマ数の計算（テーブル参照） |
| **`Reel`** | 個別リール | 回転アニメーション、シンボルの描画とループ処理 |
| **`SoundFX`** | サウンド | Web Audio APIによる効果音・BGMの合成と再生 |

---

## 2. 内部抽選ロジック

「レバーON」の瞬間に、JavaScriptの乱数を用いて「そのプレイの運命（フラグ）」を決定します。

### 抽選アルゴリズム
1. **分母の設定**: 全体を `65,536` (16bit) の範囲として扱います。
2. **テーブル選択**:
   - `NORMAL`: 通常時
   - `BONUS_ACTIVE`: ボーナス内部中（ボーナス以外の小役を抽選）
   - `BONUS_GAME`: ボーナス中（ブドウを高確率で抽選）
3. **累積重み判定**: 乱数値が各役の `weight` の累積範囲に収まるか順にチェックします。
   - 例: `FREEZE` (64) -> `REPLAY` (9298) -> ... -> `BLANK` (ハズレ)

---

## 3. 出目制御（スベリ制御）の深層

本システムの最も重要なロジックです。停止ボタンが押された瞬間のリール位置と、成立フラグに基づいて停止位置を決定します。

### スベリ計算のステップ
1. **停止ボタン押下**: 押下された「コマ（フレーム）」の位置をキャプチャ。
2. **引き込み（Pull-in）**:
   - `STOP_CONTROL_TABLE` を参照し、成立フラグに対する「ターゲット絵柄」を特定。
   - 最大4コマ（規定のスベリ範囲）の間にターゲット絵柄がある場合、その絵柄が中段（または規定ライン）に止まるようスベリコマ数を算出。
3. **蹴り（Kicking / Safe Slip）**:
   - フラグがハズレの場合、あるいは成立フラグ以外の役が揃ってしまう位置にある場合、それらを「避ける」スベリを行います。
   - `findSafeSlip` メソッドが、停止候補位置からの「入賞判定（仮想評価）」を行い、成立フラグ以外の入賞が発生しない最初の位置を探します。

## 4. ボーナス・システム詳細

実機の挙動に合わせた特殊な状態遷移を実装しています。

### ボーナス入賞（777 / 77BAR）
- 入賞時は **「リプレイ扱い」** となります（`this.state.bet = 3`）。
- メダルを消費（払い出し）せず、次ゲームのレバーONが自動的に許可されます。

### ボーナスゲーム（小役ゲーム）
- **1枚掛け固定**: ボーナス中は `MAX BET` ボタンを押すと1枚だけ消費されます。
- **24枚払い出し**: `BONUS_GRAPE_PAYOUT` により、ブドウ入賞時の払い出しを強化（通常7枚 → 24枚）。
- **規定入賞回数**: 
  - `BIG`: 12回のブドウ入賞で終了。
  - `REG`: 6回のブドウ入賞で終了。

## 5. 演出・サウンド仕様

### GOGO告知
- **先告知**: レバーON時に、内部ボーナス成立時の25%の確率で点灯。
- **後告知**: 全リール停止後の第3停止ボタン離し時（本システムでは停止時）に点灯。
- **BLINK状態**: 1G連確定時やプレミアムフリーズ時は、ランプが高速に点滅（`CSSアニメーション`）。

### BGM合成
- `Web Audio API` の `OscillatorNode`（Square波）を用いて、ボーナス専用の arpeggio 旋律を動的に生成・ループ再生します。

---

## 6. 保存と復元 (LocalStorage)

ブラウザの `localStorage` キー `slot_game_state` に、以下のステートを JSON 形式で保存します。
- 経済状態: `yen`, `credits`
- 統計情報: `spins`, `bigs`, `regs`
- ボーナス進捗: `isBonusGame`, `bonusGamesRemaining`, `internalBonus`

これにより、ボーナスゲームの途中でブラウザを閉じても、再開時に続きからプレイすることが可能です。
