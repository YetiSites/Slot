# Web Slot Machine - JS設計ドキュメント

このドキュメントでは、本スロットマシンのJavaScript（`script.js`）における設計構造、抽選システム、および出目制御ロジックの詳細について解説します。

## 1. 全体アーキテクチャ

本システムは、モダンなES6クラスベースのオブジェクト指向設計を採用しており、各機能が疎結合になるよう設計されています。

### クラス構成
- **`SlotGame`**: メインエンジン。ゲームの状態管理（クレジット、ボーナスフラグ、UI同期）とライフサイクルを統括します。
- **`ReelController`**: 3つのリールを束ねて制御する司令塔。回転・停止の同期、および「スベリ制御」の計算を担当します。
- **`Reel`**: 単一のリールのオブジェクト。自身の回転位置、速度、描画（DOM操作）を管理します。
- **`SoundFX`**: Web Audio APIを用いた音声合成エンジン。効果音の再生をカプセル化しています。

---

## 2. 内部抽選システム

抽選は「レバーON」のタイミングで実行され、そのプレイでの「成立役（フラグ）」を決定します。

### 確率テーブル (`LOTTERY_TABLE`)
全ての役は、65,536分率（16bit）の重み付けで管理されています。
- **通常状態 (`NORMAL`)**: 全ての小役およびボーナスの抽選。
- **ボーナス成立中 (`BONUS_ACTIVE`)**: すでにボーナス内部中（未入賞）の場合、小役のみを抽選し、ボーナスの重複抽選を避けます。

### 特殊演出のトリガー
- **プレミアムフリーズ**: 抽選結果が特定のフラグかつ条件を満たした場合、`isFreezing` 状態に移行し、4秒間のフリーズ演出を挿入します。
- **1G連**: `pendingBigs` 変数により、ボーナス終了後の次ゲームで強制的にBIGを成立させる仕組みを搭載しています。

---

## 3. 出目制御ロジック (スベリ制御)

本システムの核心部であり、「成立したフラグ」と「ボタンを押した位置」から、最大4コマ（本システムでは調整可能。デフォルト8フレーム分程度）のスベリを計算します。

### 制御の仕組み
1. **フラグ参照**: 押下されたリールと現在の成立フラグに基づき、`STOP_CONTROL_TABLE` から引き込み優先順位を取得します。
2. **引き込み計算**:
   - 押下位置からスベリ限界までの範囲内に、指定された「引き込み絵柄」があるか探索します。
   - 優先順位が高い位置（通常はリールの中段）に止まるよう最適なスベリコマ数を算出します。
3. **蹴り制御 (Safe Slip)**:
   - フラグが「ハズレ」の場合、あるいは成立フラグ以外の役が揃ってしまう位置（誤入賞）を避けるため、`findSafeSlip` メソッドが「何も揃わない位置」まで自動的にリールを滑らせます。
4. **ボーナス引き込み**:
   - ボーナスフラグ成立中は、どのリールでもボーナス絵柄を最大限引き込もうとします。

---

## 4. 入賞判定システム

リールが完全に停止した後、`evaluateWin` 関数が現在の出目を評価します。

- **マトリクス評価**: 3x3のシンボルマトリクスを生成し、5つの有効ライン（横3本、斜め2本）をスキャンします。
- **優先順位付き判定**: 
  1. ボーナス入賞 (777 / 77BAR)
  2. レア小役 (BAR揃い)
  3. 一般小役 (ベル、ブドウ、リプレイ)
  4. チェリー (左リール枠内判定)
- **純粋関数設計**: DOMの状態に依存せず、停止インデックスの配列から結果（払い出し枚数、役名）を返すピュアなロジックになっています。

---

## 5. UI/UX 演出

- **GOGOランプ**: `gogoState`（OFF / ON / BLINK）によって管理。先告知（レバーON時）と後告知（全リール停止時）の両方をサポート。
- **デジタルフォント**: クレジットやゲーム数の表示にはデジタルセグメント風のフォントを適用。
- **レスポンシブ**: CSSのトランスフォームとJSのピクセル計算を組み合わせ、スムーズな回転アニメーションを実現。

---

## 6. 拡張ガイド

- **確率を変える**: `LOTTERY_TABLE` の `weight` を調整します。
- **リール配列を変える**: `REEL_ARRAYS` を編集します（各リール21コマ）。
- **新しい役を追加する**: `SYMBOLS` に定義を追加し、`evaluateWin` の判定ロジックに条件を追加します。
